# Migrations folder

This folder is intentionally empty in the source you downloaded — EF Core migration
files are generated by tooling and shouldn't be hand-written.

Generate the initial migration locally (one-time):

```
dotnet tool install --global dotnet-ef   # if you don't already have it
cd InventoryFlow
dotnet ef migrations add InitialCreate
dotnet ef database update
```

The app also calls `context.Database.Migrate()` automatically on startup
(see Program.cs), so once the migration exists, just running the app will
create/update the database and seed demo data.
